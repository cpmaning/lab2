1000 Floor Building Elevator Design

    Designing an interface for an elevator for a one thousand floor building is quite tough
    since there are a lot of variables to be considered. Since this is a thousand floor building, it
    is to be expected to be a super building. There is a huge chance that this building can be
    considered a city. Where it has a residential area and commercial areas. The elevator will be
    considered as the mode of transportation inside the building.

    So, my design for the interface of the elevator is that it will be like a calculator where the user/s
    can add one floor, five floors, ten floors, twenty, fifty and a hundred floors. The interface will
    also have a clear button, so they can clear the digit they wanted to erase. Even though with this
    interface it is still hard and can have a lot of problems. One of the problems is that even with a
    calculator interface, adding one floor to a thousand can be quite problematic. Specially if it’s
    rush hour, trying to input the floor you wanted can still be a hassle. Another problem is that
    having an elevator go to one floor to one thousand can take some time.

    To solve that problem, I ve researched about some designs to help me solve this problem and I’ve
    come up with a solution. Let’s say that this building has a residence area and every fiftieth
    floor is the residence area. Every elevator will have to stop every fiftieth floor, to make it
    convenient and faster. There will be another set of elevators on the floor fifty-one up to a
    hundred floor. A calculator interface will also be implemented here since fifty is still a huge
    number but quite reasonable for a calculator interface. Here’s the visual representation of the
    interface I ve been thinking.




    It is designed to be like a calculator, but it will only take input up to two digits and specifically
    up to fifty only. Since the elevators will stop at every fiftieth floor. The buttons are designed to
    have a carving for the blind user/s, for them to be able to use the elevator. At the right part of
    the interface is the display of the floors that has been input. The interface also has an
    emergency button to click, which can be seen at the top left corner, which can be used
    whenever there is an emergency. At the top left corner there will be a speaker, which will tell
    the people what floor they are right now, just like in the trains whenever they reached a station
    a speaker will tell them the station they are right now. This will be helpful for the blind people.

    Another design of the system is that whenever a person/people comes in before it goes up or down.
    The user will need to input a floor before the elevator starts to move. The elevator can tell
    whether someone got in because of the addition of weight.

    I m sure that since it’s a very tall and large building, not everyone will explore the floors. So
    this would be my interface for the elevator for a thousand floor building.
